# Development Setup Guide

Complete guide to setting up your development environment for NexusNews.

## Prerequisites

### Required Software

1. **Android Studio** (Giraffe or later)
   - Download: [developer.android.com/studio](https://developer.android.com/studio)
   - Recommended: Latest stable version

2. **Java Development Kit (JDK) 11**
   - Android Studio includes this, or download from [adoptium.net](https://adoptium.net/)

3. **Git**
   - Download: [git-scm.com](https://git-scm.com/)

### Optional Software

- **Postman** or **Insomnia** - For testing APIs
- **DB Browser for SQLite** - For inspecting Room database

## System Requirements

- **OS**: Windows 10/11, macOS 10.14+, or Linux
- **RAM**: 8 GB minimum (16 GB recommended)
- **Disk Space**: 10 GB free space minimum
- **Display**: 1280 x 800 minimum resolution

## Step 1: Clone the Repository

```bash
git clone https://github.com/undead2146/NexusNews.git
cd NexusNews
```

## Step 2: Configure API Keys

1. Create a `local.properties` file in the project root (if it doesn''t exist):

```properties
# Android SDK location (auto-generated by Android Studio)
sdk.dir=/path/to/Android/sdk

# API Keys (add these lines)
newsapi.key=YOUR_NEWSAPI_KEY_HERE
guardian.api.key=YOUR_GUARDIAN_KEY_HERE
openrouter.api.key=YOUR_OPENROUTER_KEY_HERE
```

2. Get your API keys:
   - **NewsAPI**: Register at [newsapi.org](https://newsapi.org)
   - **Guardian API**: Register at [open-platform.theguardian.com](https://open-platform.theguardian.com/access/)
   - **OpenRouter**: Register at [openrouter.ai/keys](https://openrouter.ai/keys)

> **Note**: `local.properties` is gitignored and will NOT be committed to version control.

## Step 3: Open Project in Android Studio

1. Launch Android Studio
2. Select "Open an Existing Project"
3. Navigate to the cloned `NexusNews` directory
4. Click "OK"

Android Studio will:
- Download Gradle dependencies (first time may take 5-10 minutes)
- Index project files
- Configure SDK versions

## Step 4: Sync Gradle

If Gradle doesn''t sync automatically:

1. Click **File** → **Sync Project with Gradle Files**
2. Wait for sync to complete
3. Check for any errors in the **Build** tab

## Step 5: Configure Android SDK

1. Open **Tools** → **SDK Manager**
2. Ensure these are installed:
   - **Android SDK Platform 36** (API Level 36)
   - **Android SDK Build-Tools** (latest version)
   - **Android Emulator**
   - **Intel x86 Emulator Accelerator (HAXM)** (for Windows/Mac)

## Step 6: Set Up Emulator or Device

### Option A: Android Emulator

1. Open **Tools** → **Device Manager**
2. Click **Create Device**
3. Select a phone model (e.g., Pixel 6)
4. Choose system image (API 36 recommended)
5. Click **Finish**

### Option B: Physical Device

1. Enable **Developer Options** on your Android device:
   - Go to **Settings** → **About Phone**
   - Tap **Build Number** 7 times
2. Enable **USB Debugging**:
   - **Settings** → **Developer Options** → **USB Debugging**
3. Connect device via USB
4. Accept USB debugging prompt on device

## Step 7: Build the Project

### Using Android Studio

1. Click **Build** → **Make Project** (or `Ctrl+F9` / `Cmd+F9`)
2. Wait for build to complete
3. Check for errors in **Build** tab

### Using Command Line

```bash
# Windows
.\gradlew build

# macOS/Linux
./gradlew build
```

## Step 8: Run the App

### Using Android Studio

1. Select your device/emulator from the device dropdown
2. Click **Run** (green play button) or press `Shift+F10`
3. App will install and launch

### Using Command Line

```bash
# Install debug APK
.\gradlew installDebug  # Windows
./gradlew installDebug  # macOS/Linux
```

## Troubleshooting

### Gradle Sync Fails

**Problem**: "Could not resolve dependencies"

**Solution**:
1. Check internet connection
2. Invalidate caches: **File** → **Invalidate Caches / Restart**
3. Delete `.gradle` folder and sync again

### Build Errors: "SDK location not found"

**Problem**: Android SDK path not configured

**Solution**:
1. Open `local.properties`
2. Add: `sdk.dir=/path/to/Android/sdk`
3. Restart Android Studio

### Emulator Won''t Start

**Problem**: HAXM or virtualization issues

**Solution**:
1. Check BIOS virtualization settings (VT-x/AMD-V enabled)
2. Reinstall Intel HAXM from SDK Manager
3. Or use ARM-based emulator image

### API Keys Not Working

**Problem**: 401 Unauthorized errors

**Solution**:
1. Verify keys in `local.properties`
2. Rebuild project to regenerate BuildConfig
3. Check API key validity on provider websites

## Verify Installation

Run the test suite to ensure everything is set up correctly:

```bash

./gradlew test  # macOS/Linux

# Run instrumentation tests (requires emulator/device)
.\gradlew connectedAndroidTest
```
